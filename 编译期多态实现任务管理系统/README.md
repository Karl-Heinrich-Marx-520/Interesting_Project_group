# C++ 命令行任务管理系统
一个基于 **C++11** 开发的轻量级命令行任务管理工具，集成任务增删改查、数据持久化、日志记录功能，采用**单例模式**、**CRTP 静态多态**设计，高性能且易于扩展。

## ✨ 核心特性
- 📝 **完整任务操作**：支持添加、删除、更新、查看任务
- 🏷️ **任务属性**：包含描述、优先级（1-高/2-中/3-低）、截止日期（YYYY-MM-DD）
- 📂 **数据持久化**：任务自动保存到本地文件，重启程序不丢失
- 📜 **日志系统**：单例模式实现全局日志记录，操作可追溯
- 🚀 **高性能设计**：CRTP 奇异递归模板模式实现命令接口，无运行时多态开销
- 🎯 **灵活排序**：支持按 ID / 优先级 / 截止日期排序查看任务
- 🛠️ **易扩展**：模块化设计，新增命令无需修改核心逻辑

## 🎯 设计模式
1. **单例模式 (Singleton)**：日志系统全局唯一，线程安全
2. **CRTP (奇异递归模板模式)**：静态多态实现命令接口，提升性能
3. **命令模式 (Command)**：解耦命令调用与执行逻辑
4. **面向对象封装**：任务管理、日志、命令、数据存储完全解耦

## 📖 使用指南
程序启动后，输入以下命令操作任务：

| 命令 | 用法格式 | 功能说明 |
|------|----------|----------|
| `add` | `add <描述>,<优先级>,<截止日期>` | 添加新任务 |
| `delete` | `delete <任务ID>` | 删除指定 ID 的任务 |
| `update` | `update <ID>,<描述>,<优先级>,<截止日期>` | 更新指定任务 |
| `list` | `list [排序方式]` | 查看任务（0-ID/1-优先级/2-日期） |
| `exit` | `exit` | 退出程序 |

### 使用示例
```bash

> add 购买日用品,2,2025-04-20

ID: 2, 描述: 购买日用品, 优先级: 2, 截止日期: 2025-04-20

退出程序。
```

## 🔍 核心设计详解
### 1. 单例模式 - 日志系统
- 保证全局唯一日志实例，避免多文件同时写入冲突
- 线程安全（`std::mutex` 互斥锁）
- 自动追加写入日志，程序退出自动关闭文件

### 2. CRTP 模式 - 命令接口
**静态多态 vs 动态多态（虚函数）**
| 特性 | CRTP (本项目) | 虚函数继承 |
|------|---------------|------------|
| 多态类型 | 编译期静态多态 | 运行期动态多态 |
| 性能开销 | 零开销，支持内联 | 虚表指针 + 寻址开销 |
| 适用场景 | 编译期确定命令类型 | 运行时动态加载命令 |
| 代码复杂度 | 模板语法，编译期检查 | 语法简单，直观 |

本项目使用 CRTP 追求**极致性能**，适合命令类型固定的命令行工具。

## 🛠️ 扩展指南
如需新增命令（如 `clear` 清空任务）：
1. 在 `Command.h` 中继承 `Command<T>` 实现新命令类
2. 实现 `executeImpl` 方法
3. 在 `main.cpp` 中注册命令到映射表
4. 无需修改其他模块代码

# 📊 代码架构与逻辑调用详解
## 0. 核心设计原则
本项目遵循**单一职责原则**，将系统拆分为 4 大独立模块，模块间通过接口调用协作，无强耦合，便于维护和扩展。
**核心调用链**：`用户输入 → main(命令调度) → Command(命令执行) → TaskManager(业务逻辑) → Logger(日志) + DataStore(持久化)`

---

## 1. 各源码文件功能明细
| 文件 | 核心职责 | 依赖关系 |
|------|----------|----------|
| `Logger.h / Logger.cpp` | **单例日志系统**：全局唯一日志实例，记录所有操作、写入`log.txt`、线程安全 | 无外部依赖（C++标准库） |
| `Task.h` | **任务数据结构**：定义任务属性（ID/描述/优先级/日期），提供格式化输出 | 无外部依赖 |
| `TaskManager.h / .cpp` | **任务核心管理器**：任务增删改查、排序、文件读写（`tasks.txt`）、调用日志 | 依赖 `Task.h` + `Logger.h` |
| `Command.h` | **CRTP命令系统**：静态多态命令接口，封装`add/delete/update/list`具体命令 | 依赖 `TaskManager.h` + `Logger.h` |
| `main.cpp` | **程序入口**：处理命令行输入、命令映射、调度执行、程序主循环 | 依赖所有模块 |

---

## 3. 完整程序执行流程（从启动到退出）
### 第一步：程序初始化
1. `main()` 启动 → 创建 `TaskManager` 对象
2. `TaskManager` 构造函数自动调用 `loadTasks()` → 从 `tasks.txt` 加载历史任务
3. 初始化所有命令对象（`AddCommand`/`DeleteCommand`等），存入命令映射表
4. 打印欢迎信息，进入命令等待循环

### 第二步：处理用户命令
1. 读取用户输入（如 `add 任务,1,2025-05-01`）
2. 拆分 **命令名** 和 **参数**
3. 在命令映射表中找到对应命令对象
4. 调用命令的 `execute()` 方法执行业务逻辑
5. 执行完成后，等待下一条输入

### 第三步：程序退出
输入 `exit` → 退出循环 → 程序结束 → 所有资源自动释放

## 4. 核心功能调用链路（代码如何相互调用）
### 功能1：添加任务（add）
```
用户输入 add → main.cpp → AddCommand.executeImpl()
→ 解析参数 → TaskManager.addTask()
→ 创建Task对象 → 加入任务列表
→ Logger.log() 记录日志 → TaskManager.saveTasks() 写入文件
→ 返回执行成功
```

### 功能2：删除任务（delete）
```
用户输入 delete → main.cpp → DeleteCommand.executeImpl()
→ 解析任务ID → TaskManager.deleteTask()
→ 查找并删除任务 → Logger.log() 记录日志
→ TaskManager.saveTasks() 更新文件 → 返回结果
```

### 功能3：查看任务（list）
```
用户输入 list → main.cpp → ListCommand.executeImpl()
→ 传入排序参数 → TaskManager.listTasks()
→ 按ID/优先级/日期排序 → 控制台打印所有任务
```

### 功能4：更新任务（update）
```
用户输入 update → main.cpp → UpdateCommand.executeImpl()
→ 解析ID+新参数 → TaskManager.updateTask()
→ 修改任务属性 → Logger.log() 记录更新日志
→ TaskManager.saveTasks() 保存到文件 → 返回成功
```

## 5. 关键技术点逻辑说明
### ① 单例模式（Logger）
- 全局只能有一个日志对象，避免多线程/多模块同时写日志冲突
- 调用方式：`Logger::getInstance().log("信息")`
- 所有模块修改任务时，**主动调用Logger记录操作**

### ② CRTP静态多态（Command）
- 基类：`Command<Derived>`（模板类）
- 派生类：`AddCommand/DeleteCommand` 等
- 调用逻辑：`execute()` → 静态转换为派生类 → 调用 `executeImpl()`
- 无虚函数开销，编译期确定调用逻辑，高性能

### ③ 数据持久化（TaskManager）
- 加载：程序启动时 `loadTasks()` 读取 `tasks.txt` → 还原任务列表
- 保存：任务增删改后 **自动调用 `saveTasks()`** → 覆盖写入文件
- 无需手动保存，数据永久不丢失

### ④ 命令映射（main.cpp）
- 使用 `unordered_map` 存储 `命令名 → 命令对象`
- 新增命令只需添加映射关系，**无需修改主循环逻辑**
- 支持动态扩展命令
